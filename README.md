# 慢病用药小管家

## 项目简介

这是一个电脑网页优先的慢病用药管理应用。它帮助用户把今日用药、药品库存、补药计划、复诊安排和用药记录集中在一个页面里，减少忘记吃药、忘记买药、忘记复诊的问题。

## 技术架构

- 使用 React + Vite 构建单页网页应用。
- 页面数据默认来自内置示例数据，用户修改后保存到浏览器本地。
- 用纯规则计算药品剩余天数、补药优先级、复诊状态和 AI 风格摘要。
- 不接真实医院、药房、短信或 AI 接口，适合作为评测演示作品。

## 本地运行方法

```bash
npm install
npm run dev
```

打开终端提示的本地链接即可访问网页。

## 部署方法

```bash
npm run build
```

构建结果会生成在 `dist` 目录，可部署到任意静态网页托管平台。

## 作品提交注意事项

- 评测平台需要提交“可访问的作品链接”，不能提交本机 `localhost` 链接。
- 提交前先把项目部署到能公开访问的平台，再复制部署后的访问链接。
- 作品链接通常只能提交一次，提交前请先打开链接检查页面是否能正常访问。
- 后续如果继续修改作品，需要重新运行 `npm run build` 并重新部署，原提交平台才会看到最新版本。
- 如果使用 NoCode 平台提交，需要确认登录手机号与招聘系统一致；如果使用其他 AI 编程平台，只要最终链接能访问即可。

## 测试方法

```bash
npm test
npm run build
```

`npm test` 用于验证补药、漏服、复诊和页面状态规则；`npm run build` 用于确认网页可以正常打包。

## 已完成功能

- 总览仪表盘：今日待办、已完成、库存风险、复诊倒计时。
- 今日用药：按时间展示药品，可点击打卡并同步扣减库存。
- 药品管理：新增、编辑、删除药品，展示库存和剩余天数。
- 补药计划：根据库存和每日用量自动计算补药优先级。
- 复诊管理：添加下一次复诊，展示复诊倒计时和历史记录。
- 用药记录：展示近 7 天完成率和漏服次数。
- 购药记录：记录购买数量并自动增加库存。
- AI 健康摘要：根据库存、漏服和复诊时间生成自然语言提醒。
- 本地保存：刷新页面后保留用户修改的数据。
